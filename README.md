# 🛡️ Enterprise Security & GRC Home Lab 
### Hybrid Multi-OS Environment for Risk Assessment & Vulnerability Management

## 📌 Executive Summary
This lab serves as a localized "Enterprise Sandbox" designed to simulate a corporate network environment. As an **MBA in IT Management** and **CompTIA Security+ ce / EC-Council Certified Ethical Hacker (CEH)**, I utilize this environment to bridge the gap between technical vulnerability discovery and business-level risk mitigation.

---

## 🏗️ Lab Architecture
The environment is hosted on **Oracle VirtualBox**, utilizing a segmented network topology to ensure safe testing and isolation from the host machine.

### 🖥️ Virtual Machine Inventory (Click to view details)
*   **[🐉 Kali Linux (Attacker Node)](./Kali)** - Primary offensive security platform for penetration testing and vulnerability research.
*   **[🦜 Parrot OS (Security/Dev)](./Parrot)** - Forensic and development environment for secure tool testing.
*   **[🪟 Windows 11 (Hardened Target)](./Windows11)** - Enterprise endpoint for testing GPOs and security policy enforcement.
*   **[🐧 Ubuntu Server (Managed Endpoint)](./Ubuntu)** - Linux infrastructure simulator for auditing logs and firewall (UFW) configurations.

---

## 🚀 Key GRC & Security Use Cases
1.  **Vulnerability Management:** Running scans across the network to identify and document "High" and "Critical" risks.
2.  **Policy Compliance:** Benchmarking installations against **NIST** or **CIS Benchmarks** to ensure a hardened baseline.
3.  **Governance & Risk Reporting:** Translating technical scan data into Executive Risk Reports, simulating a **vCISO** board presentation.
4.  **Network Hardening:** Configuring internal firewalls and auditing Identity Access Management (IAM).
